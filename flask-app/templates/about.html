Marks Information

Points Values

Marks are issued depending on how many items the user has out, and for how long they have kept these items overdue.

Points Description
1 Per Day Overdue Item (per item)
5 Item damage, more serious rule violations
10 Rule Violations, Profanity, Verbal Abuse
15 Theft, violence, threatening
20 Destruction on property, serios infractions
30 Maximum amount of marks per semester. User is banned for the semester
999 Enter '999' marks to permanently ban a user in the system. This should be done using a 'New Marks' form on the dashboard
What to Know

New Marks Should be Issued from the 'Dashboard' tab. After marks have been entered, the marks can be seen from the 'Marks' tab by searching the user. A user can have multiple marks entered for them, but there is only one associated user account for each mark.

A user can be edited by navigating to the 'Users' tab and selecting the edit symbol next to the desired user. It is important to remember that while the new marks entered will update the users current standing, it will not create new marks for the User under the 'Marks' tab. So if entering new marks, always use the new marks form.

Management Functions

The management login account has access to special management functions. The first of these functions allows for the user to reset all marks in the system. This sets all marks to zero, sets all standings to 'Good Standing', the date to the current date, and the recorded status to zero. This function does not affect users who have been marked 'PermaBan' in the system. This action cannot be undone.

The other management function allows for sorting users by their 'Recorded' status. A user can be marked as recored in the system to designate when a user has been marked as banned in the R&R System. This allows for management to keep track of who has been entered into the system for organizational purposes.

Data Dictionaries

Users

Column Description
Username The UARK username of the user
Number Marks The number of marks associated with the user
Current Standing The standing of the user. 'Good Standing', 'Facility Ban (Semester)',
'Checkout Ban (One Month)', 'PermaBan'
Recent Date The most recent date the user record was updated
Recorded A 'yes' or 'no' value that indicates if management has entered a user ban into the R&R system.
This value is only accessible through management login

Marks

Column Description
Username The UARK username of the user being issued the marks
Number Marks The number of marks associated with the marks being entered
Reason The reason for the associated marks being issued.
Date The date of the marks being issued. Automatically generated by the system.
Issuer UARK username of the employee issuing the marks.
